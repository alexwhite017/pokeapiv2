    ```jsx
    <span
      className="absolute bottom-0 right-0 translate-x-1/4 translate-y-1/4 text-[7rem] font-black leading-none text-white/15 select-none pointer-events-none tabular-nums"

    ```

    **Why these choices:**
    - `absolute bottom-0 right-0` — anchors to bottom-right corner of the `relative overflow-hidden` card div
    - `translate-x-1/4 translate-y-1/4` — shifts the text right and down so roughly the bottom-right quarter is clipped by the card edge, giving the jersey-number / trading-card serial aesthetic locked in CONTEXT.md
    - `text-[7rem]` — large enough to fill most of the card height visually; cards have `max-h-36` artwork so 7rem (112px) at this size fills the card height with the number naturally
    - `font-black` — maximum weight for a bold design element read
    - `leading-none` — removes extra line height so positioning is tight
    - `text-white/15` — opacity 15% against the type-colored gradient background; clean and legible as a design element without dominating the artwork. Per CONTEXT.md this is Claude's discretion — 15% is the right balance: visible as a design element against all type colors, invisible enough not to compete with the artwork
    - `select-none pointer-events-none` — purely decorative, no interaction
    - `aria-hidden="true"` — screen readers already see the small dex label; no need to repeat

    **Placement:** Inside the `relative overflow-hidden` card root div. After the info strip `</div>`, before the outer card root `</div>`. This stacking order means the watermark renders above the gradient background but the artwork and info strip render above the watermark (artwork div and info strip div come earlier in source, but since watermark is absolutely positioned and non-interactive, z-order works naturally — however to ensure artwork stays on top, add `relative z-10` to the artwork div and info strip div, or simply add `z-0` to the watermark span).

    **Final positioning note:** Actually the simplest and most correct approach: add the watermark span *before* the artwork div in source order (so it's layered beneath by default stacking), OR add `z-0` to watermark and ensure artwork/info strip have no z-index conflict. The recommended approach is: place watermark span *after* the small top-right label and *before* the artwork div in DOM order so it naturally sits below the artwork in stacking context (both are `absolute` or the artwork is in the normal flow — artwork being in normal flow means it paints after absolutely positioned watermark, thus on top).

    **Correct final DOM order within card root div:**
    1. Small top-right label (absolute, z-auto) — already there
