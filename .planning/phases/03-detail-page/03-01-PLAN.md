    The inner div currently has `bg-surface-raised`. Remove this background — it will now be transparent, letting the gradient show. Keep all other layout classes.

    **Result:** PokemonDetails owns the gradient and signals its type to the page via CSS variable.

    - Since the CSS variable changes when PokemonDetails mounts/unmounts, NavBar needs to listen for these changes. Use a `MutationObserver` on `document.documentElement` with `{ attributes: true, attributeFilter: ['style'] }` to detect style attribute changes.

    **Simplest correct approach:** Change NavBar's background class from `bg-surface-raised` to a conditional. When `--active-type-color` is set (non-empty string from MutationObserver state), apply `style={{ backgroundColor: 'var(--active-type-color)' }}` and remove the `bg-surface-raised` class. When not set, keep `bg-surface-raised` class and no inline style.

    **Cleanup:** Return a cleanup function from useEffect that disconnects the MutationObserver.

    This approach requires no prop changes to App.jsx — NavBar reads the CSS variable PokemonDetails sets.
